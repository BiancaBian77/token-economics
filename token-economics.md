# Token 经济学：AI 时代的商业底层逻辑

> 五层框架，从成本结构到终局博弈，帮你系统性理解 AI 产品背后的经济学。

---

## 导言：为什么你需要理解 Token 经济学

2023 年，ChatGPT 让所有人第一次意识到 AI 的威力。2026 年，AI 已经渗透到搜索、写作、编程、设计、客服、招聘等几乎所有领域。但大多数人对 AI 产品背后的经济逻辑一无所知——

**每一次你跟 AI 对话，背后都有一笔实实在在的交易在发生。**

你输入的文字被拆成"token"，AI 回复的文字也是"token"。Token 是 AI 时代的基本计量单位，就像电力的"度"、通信的"流量"。理解 token 的成本、定价、流向和博弈，就是理解整个 AI 产业的商业底层逻辑。

本教案提供一个**五层框架**，帮你从零开始系统性地理解 Token 经济学：

1. **成本结构** — 一个 token 到底花多少钱？
2. **定价模型** — AI 产品怎么向用户收钱？
3. **单位经济学** — 一个 AI 产品的真实成本全景是什么？
4. **供给侧** — 算力产业链里谁最赚钱？
5. **终局博弈** — 当 token 变成水电煤，AI 创业公司靠什么活？

无论你是创业者、产品经理、投资人，还是对 AI 行业感兴趣的普通人，这五层框架都能帮你看懂 AI 产品的钱是怎么赚的、怎么花的、最终流向了哪里。

## 第一层：Token 的成本结构

### 案例：OpenAI vs DeepSeek（中美对比）

### 什么是 Token？

一个 token 大约是 0.75 个英文单词，或 0.5 个中文字符。你每次跟 AI 对话，本质上就是在**买卖 token**——发给 AI 的是"输入 token"，AI 回复的是"输出 token"。

### Token 价格表（2026 年 3 月数据）

| 模型 | 输入 ($/百万 token) | 输出 ($/百万 token) | 定位 |
|------|:---:|:---:|------|
| **OpenAI GPT-4o** | $2.50 | $10.00 | 主力模型 |
| **OpenAI o3** | $2.00 | $8.00 | 推理模型 |
| **OpenAI GPT-5** | $1.25 | $10.00 | 最新旗舰 |
| **DeepSeek V3** | **$0.14** | **$0.28** | 主力模型 |
| **DeepSeek R1** | **$0.55** | **$2.19** | 推理模型 |

### 关键认知 1：输出比输入贵 4-8 倍

GPT-4o 的输出价格是输入的 4 倍（$10 vs $2.5）。因为生成文字比理解文字难得多——每生成一个 token，模型都要重新跑一遍所有的注意力计算。

**一个啰嗦的 AI 比一个简洁的 AI 贵好几倍。**

### 关键认知 2：中美价格差 18-36 倍

| | GPT-4o | DeepSeek V3 | 差距 |
|---|:---:|:---:|:---:|
| 输入 | $2.50 | $0.14 | **18x** |
| 输出 | $10.00 | $0.28 | **36x** |

同等能力的模型，DeepSeek 只有 OpenAI 的 1/18 到 1/36。三个原因：

1. **架构创新**：DeepSeek V3 用 MoE 架构，总参数 671B 但只激活 37B（100 人公司每次只派 5 人干活）
2. **硬件成本差异**：H800（中国特供版）+ 中国数据中心（电费、人工更低）
3. **定价策略**：DeepSeek 不依赖 API 收入盈利，可以低于成本价抢市场

### 关键认知 3：推理模型有"隐藏 token"

OpenAI o3 在给你回复前会先"内部思考"——你看不到，但按输出 token 计费。一个 500 token 的回复可能实际消耗 2500 token。**用推理模型的实际成本可能是预期的 3-5 倍。**

### 关键认知 4：Token 成本在暴跌

| 时间 | GPT-4 级别输入价格 | 累计降幅 |
|------|:---:|:---:|
| 2023.03 | $30.00/百万 | — |
| 2024.05 | $5.00 | 6x |
| 2025.07 | $2.50 | 12x |
| 2026.03 | $1.25（GPT-5 batch） | **24x** |

三年降了 24 倍。这对创业公司是双刃剑：成本持续下降，但定价也会被迫下降。

### 练习：算一笔真实的账

**场景：一个 AI 智能客服处理一次用户咨询**

- 用户问题 + 历史记录：~3,000 输入 token
- AI 回复 + 内部检索：~2,000 输出 token

| 用哪个模型 | 输入成本 | 输出成本 | 单次对话总成本 |
|-----------|:---:|:---:|:---:|
| GPT-4o | $0.0075 | $0.02 | **$0.028** |
| DeepSeek V3 | $0.0004 | $0.0006 | **$0.001** |

**一次 AI 客服对话的成本：GPT-4o 约 ¥0.2，DeepSeek 约 ¥0.007。** 极其便宜——但当你每天处理 10 万次对话时，就是 ¥2 万 vs ¥700 的日成本差异。

> **本层要点**：Token 分输入/输出，输出贵 4-8 倍；中美价差 18 倍；推理模型有隐藏成本；三年降了 24 倍。

## 第二层：Token 的定价模型

### 案例：Anthropic Claude（订阅制 vs 按量计费的博弈）

### 定价的核心矛盾

AI 产品的成本是**可变的**（用户每说一句你就花一分钱），但用户希望价格是**可预测的**（每月花多少别吓到我）。这两个需求天然矛盾。

### Anthropic 的双轨定价

**面向消费者：订阅制**

| 计划 | 月费 | 使用限制 |
|------|:---:|------|
| Free | $0 | 每天约 30 条 |
| Pro | $20 | 每 5 小时约 45 条 |
| Max 5x | $100 | 5 倍 Pro 用量 |
| Max 20x | $200 | 20 倍 Pro 用量 |

**面向开发者：按量计费**

| 模型 | 输入 ($/百万 token) | 输出 ($/百万 token) |
|------|:---:|:---:|
| Haiku 4.5 | $1.00 | $5.00 |
| Sonnet 4.6 | $3.00 | $15.00 |
| Opus 4.6 | $5.00 | $25.00 |

### 订阅制的数学本质：健身卡模型

Anthropic 的数据显示：

- 平均用户每天消耗约 $6 API 等价 → 实际推理成本约 $0.6/天 → **$18/月**
- 90% 的用户每天低于 $12 API 等价
- 只有不到 5% 触及周限额

```
大量轻度用户的利润 + 少量重度用户的亏损 = 整体微利或持平
```

跟健身房一样——卖 1000 张卡，赌大部分人办了卡不来。

### 关键发现：API 标价 ≠ 实际成本

Anthropic 的 API 有约 **10 倍加价率**。$3/百万 token 的 Sonnet，实际推理成本约 $0.3/百万 token。加价包含了：研发分摊、GPU 折旧、安全投入、利润。

所以一个"消耗 $5,000 API 等价"的重度用户，Anthropic 的实际成本约 $500——不是 $5,000。

### 四种主流定价模式

| 模式 | 代表 | 优点 | 缺点 |
|------|------|------|------|
| **纯订阅** | ChatGPT Plus $20/月 | 用户可预测 | 重度用户亏钱 |
| **纯按量** | OpenAI API | 成本精确可控 | 用户怕"惊喜账单" |
| **混合制** | Claude Max | 兼顾预测和灵活 | 定价复杂 |
| **按结果计费** | 部分 Agent 产品 | 按价值付费 | 结果难归因 |

2026 年 31% 的 AI 公司采用混合制。78% 的 IT 负责人遇到过 AI 工具的"意外账单"。

### 练习：给 AI 客服 SaaS 设计定价

基于第一层的成本（GPT-4o ¥0.2/次对话），设计三档方案：

| 方案 | 月费 | 含对话数 | 超出价 | 月 token 成本 | 毛利率 |
|------|:---:|:---:|:---:|:---:|:---:|
| Starter | ¥599 | 5,000 次 | ¥0.2/次 | ¥1,000 | **-67%（亏损）** |
| Pro | ¥2,999 | 20,000 次 | ¥0.15/次 | ¥4,000 | **-33%（亏损）** |
| Enterprise | ¥9,999 | 100,000 次 | ¥0.10/次 | ¥20,000 | **-100%（巨亏）** |

等等——全亏？因为这里用的是 GPT-4o。如果换成 DeepSeek V3：

| 方案 | 月费 | 月 token 成本 | 毛利率 |
|------|:---:|:---:|:---:|
| Starter | ¥599 | ¥35 | **94%** |
| Pro | ¥2,999 | ¥140 | **95%** |
| Enterprise | ¥9,999 | ¥700 | **93%** |

**模型选择直接决定了你是盈利还是破产。** 但 token 成本只是冰山一角——第三层告诉你为什么。

> **本层要点**：订阅制 = 健身卡逻辑；API 标价有 10 倍加价；混合制成为主流；模型选择直接决定盈亏。

## 第三层：AI 产品的单位经济学

### 案例：Google（Chat / Nano / Veo 3 的成本差异）

### COGS 回来了

传统 SaaS 的边际成本趋近于零，毛利率 80-90%。AI 产品彻底打破了这个规则——**每一次用户请求都有实实在在的推理成本。**

| 指标 | 传统 SaaS | AI 产品 |
|------|:---:|:---:|
| 毛利率 | 80-90% | 50-70% |
| 边际成本 | ~0 | 每次请求有成本 |
| 规模效应 | 越大越赚 | 越大成本越高 |

### Google 三种 AI 产品的成本拆解

**文本对话（Gemini Chat）**

```
一次对话 ≈ $0.0025（用 Flash 模型）
活跃用户每月 ≈ $3.75
Google AI Pro 订阅 = $19.99/月
→ 大部分用户赚钱
```

**端侧小模型（Gemini Nano）**

```
推理成本 = $0（跑在用户手机上）
Google 的成本 = 模型训练分摊 + 软件维护
→ 不直接赚钱，用来绑定 Android 生态
```

**视频生成（Veo 3）**

```
10 秒视频 = $4.00（标准质量）
对比一次 Chat = $0.0025
→ 视频比文本贵 1,600 倍
```

Google AI Pro 用户每月 1000 credit，一个 10 秒视频消耗 125 credit。**一个用户每月只能生成 8 个短视频就用完额度。** 重度视频用户的月成本可能是订阅费的 5-15 倍。

### 用户小王的一天

| 行为 | 模型 | 成本 |
|------|------|:---:|
| 问天气+日程 | Nano（端侧） | $0 |
| 路上聊 5 轮 | Flash | $0.012 |
| 分析 20 页 PDF | Pro | $0.18 |
| 生成一张图片 | Imagen 3 | $0.04 |
| 写一封长邮件 | Flash | $0.008 |
| 生成 8 秒视频 | Veo 3 Fast | **$1.20** |
| **全天总计** | | **$1.44** |

每月 22 个工作日：$31.68。他付了 $19.99。**亏 $11.69。**

去掉视频？月成本 $5.28。**赚 $14.71。**

**一个功能就能反转盈亏。** 这就是为什么所有 AI 产品都在高成本功能上设严格额度。

### Token 只是冰山一角

AI 产品的真实成本全景：

| 成本项 | 占比 | 说明 |
|--------|:---:|------|
| 推理 token | 15-25% | 第一层算的 |
| GPU 基础设施 | 10-15% | 闲置也要付的固定成本 |
| 模型训练分摊 | 5-10% | GPT-5 训练 $500M+，分摊到每次请求 |
| 数据/带宽 | 3-5% | 对话历史、日志 |
| 安全合规 | 3-5% | 内容审核、红队测试 |
| 研发人力 | 20-30% | AI 工程师年薪 $300K-$1M |
| 获客成本 | 15-25% | 营销、销售 |
| 一般管理 | 5-10% | 办公、法务 |

**真实毛利率：55-70%。真实净利率：-20% 到 +10%。** 大部分 AI 公司目前亏钱。

### 行业三档

| 级别 | 毛利率 | 特征 |
|------|:---:|------|
| Supernova（超新星） | ~25% | 未优化，烧钱阶段 |
| Shooting Star（流星） | ~60% | 定制模型+精细定价 |
| North Star（北极星） | 75%+ | 接近传统 SaaS（极少） |

> **本层要点**：Token 只占成本 15-25%；不同模态差 1600 倍；模型路由是提升毛利率的关键；大部分 AI 公司目前净亏损。

## 第四层：算力供给侧

### 案例：NVIDIA + 云厂商

### 钱的流向

```
用户付给产品 → 产品付给模型厂商 → 模型厂商付给云厂商 → 云厂商付给 NVIDIA
    ¥50              $0.12              $3/GPU·hr          $40,000/chip
```

### NVIDIA：食物链顶端

| 指标 | 数字 |
|------|------|
| 年收入 | $2,159 亿 |
| 数据中心占比 | 91% |
| 毛利率 | **73-75%** |
| GPU 市场份额 | 87% |

**一块 B200 芯片：制造成本 $6,000，售价 $45,000，芯片级毛利率 82%。**

对比：iPhone 毛利率 60%，星巴克拿铁 70%，NVIDIA B200 **82%**。

**NVIDIA 卖的不是芯片，是垄断性的"AI 税"。** 每一次 AI 推理都在给 NVIDIA 交税。

### 云厂商：中间商

| 云厂商 | H100 每 GPU 每小时 | 月收入 |
|--------|:---:|:---:|
| AWS | $3.90 | $2,808 |
| Google Cloud | $3.00 | $2,160 |
| Azure | $6.98 | $5,025 |

一块 H100 买入价 ~$30,000。按 AWS 价格，**11 个月回本**。但 GPU 利用率通常只有 30-50%——实际回本周期拉长到 **27 个月**。

### 模型厂商：还在亏钱

OpenAI 2025 年收入约 $130 亿，但支出 $150-180 亿。

DeepSeek 完全不同：~200 人团队（OpenAI 的 1/15），不依赖 API 盈利，可以亏损定价抢市场。

### 自部署 vs 调 API

| 月 token 用量 | 更便宜的选择 | 原因 |
|:---:|------|------|
| < 5000 万 | 调 API | 固定成本太高 |
| 5000 万-5 亿 | 看具体场景 | 灰色地带 |
| > 5 亿 | 自部署开源模型 | 节省 40-60% |

隐性成本：运维工程师 $750-3,000/月，GPU 利用率只有 30-40%，无 SLA 保障。**大多数团队在 2026 年选择调 API。**

### 谁最赚钱？

| 层级 | 代表 | 毛利率 | 护城河 |
|------|------|:---:|------|
| 芯片 | NVIDIA | 75% | 极深 |
| 云基础设施 | AWS/Azure | 50-60% | 深 |
| 模型厂商 | OpenAI/Anthropic | 60-70% | 中 |
| 应用层 | AI 创业公司 | 30-50% | 浅 |

**利润从下往上逐层递减。** 但两个力量在改变格局：开源模型侵蚀 API 利润 + AI 芯片竞争加剧。终局：token 成本趋近于电费 + 硬件折旧。

### 练习：从一次 AI 客服对话看价值分配

```
用户每月付给 AI 客服 SaaS：¥2,999
SaaS 付给 OpenAI API：~¥4,000（20,000 次 × ¥0.2）
OpenAI 付给 Azure GPU 租金：~¥60
Azure 付给 NVIDIA 芯片分摊：~¥25

注意：SaaS 公司 token 成本已经超过用户付费。
所以要么换便宜模型，要么提价，要么死。
```

> **本层要点**：NVIDIA 75% 毛利率吃最大利润；利润从底层到应用层递减；开源+芯片竞争在压低整条链；token 成本终将趋近电费。

## 第五层：终局博弈

### 案例：Manus（当 Token 变成水电煤）

### Manus 的经济学

| 指标 | 数字 |
|------|------|
| ARR | $1 亿（8 个月达到，史上最快） |
| 团队 | 78 人 |
| 人均收入 | $128 万/年/人 |
| 累计处理 token | 147 万亿 |
| 结局 | 被 Meta $25 亿收购 |

**关键数字：多 Agent 系统消耗 15 倍 token。** 但 token 在 Manus 的成本里只占 20%。真正的大头是虚拟机、浏览器自动化、存储——"让 AI 去操作真实世界"的计算资源。

### Meta 花 $25 亿买的是什么？

不是 token 差价，而是：

1. **Agent 编排系统**——经过 147 万亿 token 实战打磨的多 agent 协作框架
2. **8000 万次任务的数据飞轮**——哪些策略有效、哪些工具序列成功率高
3. **用户工作流嵌入**——用户把 Manus 嵌入日常流程后，迁移成本极高

### 终局推演：2028 年 AI 产业

假设 token 再降 10 倍，开源追平闭源，NVIDIA 垄断被打破：

| 层级 | 护城河变化 | 毛利率趋势 |
|------|----------|:---:|
| 芯片层 | 缩小（竞争加剧） | 75% → 55-65% |
| 云基础设施 | 稳定（规模效应） | 50-60% → 40-50% |
| 模型层 | **大幅缩水**（开源追平） | 60% → 20-40% |
| 中间件层 | 新出现（Agent 编排） | 50-65% |
| 应用层 | **取决于护城河深度** | 40-70% |

**最大变化：模型层利润被压缩，应用层和中间件层的相对价值上升。**

### AI 创业公司的四种护城河

| 类型 | 例子 | 可复制性 |
|------|------|------|
| **专有数据** | 千万次用户交互的模式数据 | 极难 |
| **工作流嵌入** | 用户把产品嵌入了日常流程 | 迁移成本高 |
| **领域专家系统** | 特定行业积累的知识体系 | 需多年积累 |
| **网络效应** | 双边平台的供需互相吸引 | 先发优势 |

88% 的 CEO 已经认为"部署速度"比"模型精度"更重要——市场在从"谁的模型好"转向"谁能更快解决我的问题"。

### 练习：AI 客服 SaaS 的终局推演

**2026（现在）：**
- 调 GPT-4o API，token 成本 ¥0.2/次
- 没有自有数据，护城河 = 0
- 竞争对手可以一周内做出同样的产品

**2027（如果什么都不做）：**
- 10 个竞品出现，价格战从 ¥2,999 打到 ¥599
- 毛利率从 95% 降到 30%
- 融资烧完，公司死亡

**2028（如果做对了）：**
- 积累了 500 万次真实客服对话 → 知道什么回复策略转化率最高
- 自研小模型（基于数据微调）→ 成本降到 API 的 1/10
- 深度对接客户 CRM/ERP → 替换成本极高
- "用 XX 做客服"成为行业标准 → 品牌护城河
- **从"token 经销商"变成"智能客服的事实标准"**

> **本层要点**：90%+ 的 API wrapper 公司会失败；活下来需要数据飞轮、工作流嵌入、领域知识或网络效应；终局不是做经销商，而是成为某个领域的事实标准。

| **第一层：成本结构** | 输出比输入贵 4-8x，中美价差 18x，三年降 24 倍 | OpenAI vs DeepSeek |
| **第二层：定价模型** | 订阅制 = 健身卡逻辑，混合制成为主流 | Anthropic Claude |
| **第三层：单位经济学** | Token 只占 15-25%，视频比文本贵 1600 倍 | Google Chat/Nano/Veo3 |
| **第四层：供给侧** | NVIDIA 75% 毛利率吃最大利润，利润底层到应用层递减 | NVIDIA + 云厂商 |
| **第五层：终局博弈** | Token 变水电煤后，护城河 = 数据+工作流+领域知识+网络效应 | Manus |

### 最终回答

> **所有 AI 创业公司的起点是算力经销商，但终点不能是。**

起步靠 token 差价活着没问题。但如果三年内不能建立独立于 token 的护城河，就会在 token 成本持续下降的过程中被淘汰。

Token 是 AI 时代的石油。但卖石油的不是最赚钱的——**把石油变成塑料、变成化纤、变成药品的人**，才是最终的赢家。
